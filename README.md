# 电商用户转化归因与策略分析

## 📌 项目简介

本项目基于阿里云天池平台提供的淘宝用户行为数据集，利用 SQL（MySQL）进行数据清洗与多维分析，并结合 Python（Matplotlib、Plotly）实现数据可视化。
分析旨在诊断用户从“浏览”到“支付”全链路的转化瓶颈，定位核心流失环节，并为业务方提供可落地的运营策略建议和A/B测试设计方案。

## 🔍 核心分析框架

1. **数据清洗与验证**：处理异常时间戳，定义分析口径。
2. **漏斗诊断**：构建“浏览→加购→支付”用户级转化漏斗，定位瓶颈。
3. **多维下钻**：从**时段、品类、用户分层**三个维度拆解流失原因。
4. **策略建议**：提出4项可落地的运营策略，并设计A/B测试验证方案。

## 📊 数据来源

- **来源**：阿里云天池 - 淘宝用户行为数据
- **时间范围**：2017-11-25 至 2017-12-03（共9天）
- **抽样方式**：5%随机抽样
- **数据规模**：
  - 独立用户：52.9万人
  - 浏览（pv）：89.6万次
  - 加购（cart）：5.5万次
  - 购买（buy）：2.0万次

## 🛠️ 使用工具

- **数据库**：MySQL
- **编程语言**：Python 3.x
- **可视化库**：Matplotlib, Plotly
- **版本控制**：Git

## 🔎 核心发现（摘要）

1. **转化漏斗** : 加购→支付转化率仅 **2.92%**，97%用户流失在支付环节 
2. **时段分析** : 上午10点转化率最高（3.21%），晚8-10点流量大但转化低 
3. **品类分析** : 部分品类（如800581）用户直接购买倾向显著（pv→buy达10.65%） 
4. **用户分层** : 高频购买用户（Top 20）几乎不使用“加购”功能，习惯“即看即买” 

## 💡 策略建议

1. **支付流程优化（P0）**：简化支付步骤，支持一键支付
2. **时段差异化运营（P1）**：上午主推高客单价，晚间主推种草引导
3. **品类差异化运营（P1）**：高直接购买品类优化详情页，高加购品类强化提醒
4. **用户分层运营（P2）**：高频用户缩短路径，低频用户增加触达

## 🧪 A/B测试设计

针对策略1（支付流程简化），设计了完整的A/B测试方案：
- **分流方式**：用户ID末位随机分流（对照组/实验组各50%）
- **核心指标**：加购→支付转化率
- **预期提升**：2.92% → 4.50%（+54%相对提升）
